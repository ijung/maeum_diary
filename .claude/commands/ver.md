새 버전으로 올리고 Git 태그를 푸시한다.

## 절차

1. 사용자에게 새 버전을 묻는다. 현재 버전은 `pubspec.yaml`의 `version` 필드에서 읽는다.
   - 버전 형식: `{major}.{minor}.{patch}+{build}` (예: `1.0.7+7`)
   - 사용자가 버전 문자열만 입력할 수도 있고, "패치 올려줘" 같은 자연어로 말할 수도 있다.
   - 사용자가 아무것도 입력하지 않고 엔터를 치면, patch와 build 번호를 각각 1씩 올린 버전을 자동으로 사용한다. (예: `1.0.6+6` → `1.0.7+7`)

2. `pubspec.yaml`의 `version` 필드를 새 버전으로 수정한다.

3. 변경 사항을 커밋한다.
   - 커밋 메시지: `chore: 앱 버전 업데이트 ({이전버전} → {새버전})`

4. Git 태그를 생성하고 푸시한다.
   - 태그명: `v{major}.{minor}.{patch}` (빌드 번호 제외, 예: `v1.0.7`)
   - `git tag v{버전} && git push origin v{버전}`

5. 완료 후 새 버전과 태그를 사용자에게 알린다.

6. 현재 버전에 대한 500자 내의 출시 노트를 아래 템플릿에 맞쿼 출력한다.

```
✨ 새로운 기능
- [추가된 기능 간단 설명]

🛠 개선 사항
* [UI/UX 개선 내용]
* [성능 개선 내용]

🐛 버그 수정
* [수정된 문제 간단 설명]

항상 더 나은 경험을 위해 노력하고 있습니다.
```

## 주의사항

- 태그 푸시 전에 사용자에게 확인을 받는다.
- 이미 같은 태그가 존재하면 사용자에게 알리고 중단한다.
