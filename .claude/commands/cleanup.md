전체 코드를 대상으로 클리닝, 테스트 보완, 라이브러리 최신화를 수행한다.

## 주의사항

- dart mcp 도구(`mcp__dart__analyze_files`, `mcp__dart__run_tests`, `mcp__dart__pub`)를 shell 명령 대신 사용한다.
- 각 단계는 순서대로 진행하며, 이전 단계의 문제가 해결된 후 다음 단계로 넘어간다.

## 절차

### 1단계: 코드 클리닝

전체 `lib/` 코드를 대상으로 아래 항목을 검토하고 수정한다.

- **중복 코드**: 동일 로직이 여러 곳에 있으면 공통 유틸로 추출
- **불필요한 주석**: 코드 자체로 이해 가능한 what 설명 주석 제거 (why 주석은 유지)
- **비활성화 상태 이중 처리**: Flutter 위젯이 자동 처리하는 것을 수동으로 중복 처리하는 경우 제거
- **미사용 변수/import**: 사용하지 않는 선언 제거
- **문자열 상수**: 동일 문자열이 여러 곳에 하드코딩된 경우 상수로 추출
- **Hexagonal Architecture 경계 위반**: Presentation → Domain 직접 의존 등 레이어 역방향 의존 수정

### 2단계: 테스트 커버리지 보완

`test/` 디렉터리의 기존 테스트를 검토하고 누락된 케이스를 추가한다.

- 정상 흐름과 예외/실패 케이스가 모두 존재하는지 확인
- UseCase, Repository, Domain Value Object 위주로 커버리지 확인
- 구현이 아닌 동작(Behavior) 중심으로 작성
- 테스트 파일 경로: `test/` (기존 구조 유지)

### 3단계: 라이브러리 최신화

`mcp__dart__pub` 도구로 의존성을 최신화한다.

```
mcp__dart__pub(args: ["upgrade"])
```

업그레이드 후 코드에 breaking change가 발생하면 수정한다.

### 4단계: 테스트 실행

모든 테스트가 통과할 때까지 반복 실행한다.

```
mcp__dart__run_tests()
```

실패한 테스트가 있으면 원인을 분석하고 수정 후 재실행한다.

### 5단계: 정적 분석

lint 경고가 없을 때까지 반복 실행한다.

```
mcp__dart__analyze_files(file_paths: ["lib/", "test/"])
```

경고가 있으면 수정 후 재실행한다.

### 6단계: CLAUDE.md 검토

`CLAUDE.md`를 읽고 현재 코드 상태와 맞지 않는 내용이 있으면 수정한다.

- 아키텍처 설명이 실제 레이어 구조와 일치하는지
- 기술 스택 항목이 `pubspec.yaml`의 실제 의존성과 일치하는지
- 테스트 구조 설명이 `test/` 디렉터리 실제 파일 목록과 일치하는지
